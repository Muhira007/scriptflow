Dua masalah ini terjadi karena dua hal yang berbeda. Berikut adalah penjelasan dan solusinya:

### 1. Masalah Middleware Deprecated
**Penyebab:** Pada versi Next.js terbaru yang Anda gunakan (v16.x), aturan penamaan untuk berkas *middleware* sudah diubah dari `middleware.ts` menjadi `proxy.ts`.
**Solusi:** Saya sudah langsung menjalankan perintah untuk me-*rename* (mengganti nama) berkas `middleware.ts` menjadi `proxy.ts`. Error ini seharusnya sudah hilang sekarang.

### 2. Error Database `password authentication failed`
**Penyebab:** Saat ini aplikasi (serta perintah `db:push` dan `db:seed`) mencoba terhubung ke database dengan detail dari file `.env`:
`DATABASE_URL=postgresql://user:password@localhost:5432/vogen`

Karena URL di atas hanya *placeholder/dummy*, sistem Anda menolaknya (tidak ada PostgreSQL lokal dengan *username* "user" dan *password* "password"). 

**Solusi:** Anda memiliki 2 pilihan untuk mengatasi ini:
*   **Pilihan A (Gunakan PostgreSQL Anda sendiri):** Buka file `.env` dan ubah `DATABASE_URL` sesuai dengan kredensial PostgreSQL yang ada di komputer Anda (atau gunakan layanan gratis seperti [Neon.tech](https://neon.tech)).
*   **Pilihan B (Ganti ke SQLite):** Jika Anda tidak ingin pusing menginstal/mengonfigurasi PostgreSQL, **saya bisa mengubah sistem ini menjadi menggunakan SQLite (Database berbasis file lokal)**. Dengan SQLite, sistem akan berjalan 100% secara instan di komputer Anda tanpa konfigurasi *server* apa pun.

Apakah Anda ingin saya mengubah proyek ini ke **SQLite** saja agar bisa langsung dijalankan?
